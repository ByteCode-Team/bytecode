ByteCode Documentation

Welcome to ByteCode! This guide will help you get started and make the most of your new IDE.

Installation

ByteCode is available for Windows. Download the latest release from our GitHub page.

1

Download

Get the latest .exe installer from the releases page.

2

Install

Run the installer and follow the prompts.

3

Launch

Open ByteCode from your Start menu or desktop shortcut.

Interface Overview

ByteCode's interface is designed to be familiar and intuitive:

  • Title Bar - Contains the menu, window controls, and current file path
  • Sidebar - File explorer for navigating your project
  • Editor - Main code editing area with syntax highlighting
  • Terminal - Integrated terminal (toggle with Ctrl+`)
  • Status Bar - Shows cursor position, language, and encoding
  • AI Panel - Chat with the AI assistant (click the button in bottom-right)

AI Setup

ByteCode supports multiple AI providers. Here's how to set them up:

Free Options (No API Key)

  • Puter.js - Default provider, works out of the box but has rate limits
  • Ollama - Run AI models locally on your machine
  • LM Studio - Another local AI option

API Key Required

  • Groq - Free tier available at console.groq.com
  • Claude - Anthropic's AI at console.anthropic.com
  • OpenAI - GPT models at platform.openai.com
  • Google Gemini - At makersuite.google.com
üí° Tip: Groq offers a generous free tier and is very fast. It's a great option if you want better performance than Puter.js without paying.

Using the AI

The AI assistant can help you in several ways:

  • Explain Code - Select code and click "Explain" to understand what it does
  • Fix Issues - Let the AI find and fix bugs in your code
  • Refactor - Improve code quality and readability
  • Add Comments - Automatically document your code
  • Chat - Ask any coding question in natural language

Agent Mode

ByteCode's AI can act as an agent, directly modifying your files:

  • Ask the AI to "create a new file called utils.js with helper functions"
  • Request changes like "add error handling to this function"
  • The AI will use tools to read, write, and modify files
  • Changes are applied automatically and the editor refreshes

Installing Extensions

Extensions add new features to ByteCode. There are two ways to install them:

Method 1: From .bcext File

  1. Go to Extensions ‚Üí Install Extension (.bcext)
  2. Select the .bcext file
  3. The extension will be installed and loaded automatically

Method 2: Manual Installation

  1. Go to Extensions ‚Üí Open Extensions Folder
  2. Create a new folder for your extension
  3. Add manifest.json and index.js
  4. Restart ByteCode

Creating Extensions

Extensions are simple JavaScript files with a manifest. Here's the structure:

manifest.json

{
    "id": "my-extension",
    "name": "My Extension",
    "version": "1.0.0",
    "description": "What my extension does",
    "author": "Your Name",
    "main": "index.js"
}

index.js

// Access the ByteCode API via the 'bytecode' object

// Add a status bar item
bytecode.ui.addStatusBarItem('my-item', 'üëã Hello', () => {
    bytecode.ui.showNotification('Hello from my extension!', 'success');
});

// Listen for events
bytecode.hooks.on('file:open', (data) => {
    console.log('File opened:', data.file.name);
});

// Use the editor
const code = bytecode.editor.getValue();
bytecode.editor.insertText('// Added by extension');

Packaging as .bcext

A .bcext file is just a ZIP archive. To create one:

cd my-extension
zip -r ../my-extension.bcext manifest.json index.js

Extension API Reference

Extensions have access to the bytecode object with these APIs:

bytecode.editor

bytecode.editor.getValue()              // Get editor content
bytecode.editor.setValue(text)          // Set editor content
bytecode.editor.getSelection()          // Get selected text
bytecode.editor.insertText(text)        // Insert at cursor
bytecode.editor.getLanguage()           // Get current language
bytecode.editor.getCursorPosition()     // {lineNumber, column}
bytecode.editor.focus()                 // Focus the editor

bytecode.files

bytecode.files.getCurrentFile()         // Get current file info
bytecode.files.getOpenFiles()           // List all open files
bytecode.files.openFile(path)           // Open a file
bytecode.files.saveCurrentFile()        // Save current file
bytecode.files.readFile(path)           // Read file content
bytecode.files.writeFile(path, content) // Write to file

bytecode.ui

bytecode.ui.showNotification(msg, type) // Show notification (info/success/error)
bytecode.ui.addStatusBarItem(id, text, onClick)
bytecode.ui.updateStatusBarItem(id, text)
bytecode.ui.removeStatusBarItem(id)
bytecode.ui.addMenuItem(menuId, {label, action})
bytecode.ui.addContextMenuItem({label, action})

bytecode.hooks

bytecode.hooks.on(event, callback)      // Subscribe to event
bytecode.hooks.off(event, callback)     // Unsubscribe

// Available events:
// 'editor:change', 'file:open', 'file:save', 'file:close'

bytecode.terminal

bytecode.terminal.execute(command)      // Run command (returns Promise)
bytecode.terminal.show()                // Show terminal
bytecode.terminal.write(text)           // Write to terminal

bytecode.ai

bytecode.ai.chat(message)               // Send message to AI (returns Promise)
bytecode.ai.getProvider()               // Get current AI provider info

bytecode.storage

bytecode.storage.get(extId, key)        // Get stored value
bytecode.storage.set(extId, key, value) // Store value
bytecode.storage.remove(extId, key)     // Remove value
